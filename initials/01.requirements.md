# System Architecture: Multi-Agent Decision Assistant

## 1. Concept Overview

A dynamic, general-purpose "Personal Board of Directors" designed to help indecisive users make choices. The system uses a conversational interface to understand the user's dilemma, autonomously researches options, builds a decision matrix, and provides an empathetic, data-driven recommendation.

## 2. Technology Stack & Deployment (< $5/month)

- Backend Framework: Google Agent Development Kit (ADK) - Python (google-adk)
- LLM Model: Gemini 2.5 Pro / Flash (via Vertex AI or Google AI Studio)
- Context Memory / Database: Google Cloud Firestore (Free Tier) to store session history, user preferences, and structured decision matrix data.
- Frontend: Vite + React (deployed for free on Firebase Hosting or Vercel).
- API Deployment: Google Cloud Run (Fully managed, scales to zero, generous free tier fits the $5 budget).

## 3. Multi-Agent Architecture (Dynamic Orchestration)

Instead of a rigid linear pipeline, the system uses a Supervisor Pattern via ADK's dynamic routing capabilities.

### The Primary Agent (The Orchestrator)
The only agent the user directly speaks to. Its job is to maintain conversational flow, detect when more information is needed, and delegate tasks to sub-agents.
- Trigger: User messages.
- Action: Uses LLM reasoning to either reply directly to the user OR seamlessly route a task to a sub-agent and wait for the result.

### The Sub-Agents (The Specialists)

#### The Interviewer:
* Role: Asks clarifying questions to define the criteria of the decision (e.g., budget, timeline, dealbreakers).

#### The Researcher:
* Role: Gathers external facts.
* Tools: Google Search Grounding (via Gemini built-in tools) to fetch up-to-date information on the available options.

#### The Evaluator & Analyst:
* Role: Compares the researched options against the user's exact criteria.
* Tools: SQLite MCP Server (to structure the pros/cons into a queryable temporary database) and a Custom Python Tool (Weighted Matrix Calculator).

#### The Supporter:
* Role: Monitors the user's sentiment and injects encouragement to prevent analysis paralysis.

## 4. Model Context Protocol (MCP) Integration Strategy

To fulfill the hackathon's MCP requirement, the system will utilize the following tools:

- Data Structuring (Evaluator): Standard SQLite MCP. The Evaluator agent uses this to dynamically create a "Decision Matrix" table, insert the options, and score them. This handles the structured data requirement perfectly.
- Knowledge Export (Primary): Notion MCP or Google Drive MCP. Once a decision is reached, the user can ask the system to "Save this report," and the agent will format the decision history and export it directly to their personal workspace.

## 5. Typical Dynamic Workflow Example

- User: "I don't know which laptop to buy for my computer science degree."
- Primary Agent routes to Interviewer, which prompts: "Let's figure this out. Do you prefer Mac or Windows? What is your maximum budget?"
- User: "Mac, around $1200 max."
- Primary Agent routes to Researcher, which uses Google Search Grounding to find current prices for M2/M3 MacBook Airs and entry-level MacBook Pros.
- Primary Agent routes to Evaluator, which uses the SQLite MCP to build a pro/con matrix weighing battery life vs. processing power for CS students within the $1200 budget.
- Primary Agent synthesizes this and presents the final, structured recommendation to the user, alongside words of encouragement from the Supporter agent.
