# Business Requirements Document (BRD)

**Project Name:** Indecisive App (Personal Board of Directors)
**Target Event:** GenAI APAC Hackathon by Hack2Skills
**Document Version:** 1.0

## 1. Executive Summary

The "Indecisive App" is a multi-agent AI system designed to help users overcome decision paralysis. Acting as a "Personal Board of Directors," the system orchestrates multiple specialized AI agents to interview the user, research options, evaluate data, and provide an empathetic, structured recommendation. This project directly answers the GenAI APAC Hackathon problem statement by demonstrating dynamic multi-agent coordination, database integration, and tool usage via the Model Context Protocol (MCP).

## 2. Problem Statement

- **User Problem:** Individuals frequently experience decision fatigue and "analysis paralysis" when faced with complex choices (e.g., buying a laptop, planning a career move). They struggle to objectively weigh pros and cons, gather accurate information, and remove emotional bias from the decision-making process.
- **Hackathon Problem:** The hackathon requires building a system that demonstrates coordination between a primary agent, sub-agents, external tools (MCP), and a database to execute a multi-step real-world workflow.

## 3. Project Objectives (SMART Goals)

- **Specific:** Build and deploy a multi-agent AI API using Google ADK (Python) that successfully routes tasks between 5 distinct agent personas.
- **Measurable:** Implement at least 2 distinct MCP tools (e.g., SQLite, Notion) and 1 grounding tool (Google Search). Keep cloud infrastructure costs strictly under $5.00 for the 1-month duration.
- **Achievable:** Utilize free-tier services (Google Cloud Run, Firestore, Vercel/Firebase Hosting) and a simplified React frontend to ensure completion within the hackathon timeframe.
- **Relevant:** Directly fulfills the Hack2Skills judging criteria regarding multi-agent coordination, structured data retrieval, and API-based deployment.
- **Time-Bound:** Complete end-to-end deployment and testing prior to the hackathon submission deadline.

## 4. Target Audience & Stakeholders

- **End-Users:** Individuals who struggle with decision-making and need structured, objective, and empathetic guidance.
- **Hackathon Judges (Hack2Skills):** Evaluating the technical architecture, multi-agent orchestration, and adherence to the prompt.
- **Development Team:** The hackathon participants building the frontend, backend API, and AI agent prompts.

## 5. Project Scope

### In-Scope:
- A conversational UI (Frontend) for users to interact with the system.
- A Primary Agent (Supervisor) capable of dynamically routing tasks.
- Specialized Sub-Agents: Interviewer, Researcher, Evaluator, Supporter.
- Google Search Grounding for real-time data retrieval.
- SQLite MCP integration for structuring comparison matrices.
- Firestore integration for saving and tracking user decision histories.

### Out-of-Scope:
- User authentication/login systems (to save time, sessions will be managed via simple session IDs).
- Voice-to-text or Text-to-voice capabilities.
- Native iOS/Android mobile applications.

## 6. Business Constraints & Risks

- **Budget Constraint:** Total infrastructure cost must not exceed $5/month. (Mitigation: Avoid strictly provisioned databases like Cloud SQL; use Serverless Cloud Run and Firestore free tiers).
- **Latency Risk:** Multi-agent systems can have high response latencies if agents run sequentially. (Mitigation: Use asynchronous API calls and show a "Thinking/Routing" UI state on the frontend).
- **API Limits:** Free tiers for LLM APIs and MCP tools may hit rate limits during heavy testing.
