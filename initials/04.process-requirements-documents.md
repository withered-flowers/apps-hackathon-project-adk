# Product Requirements Document (PRD)

**Project Name:** Decidely.ai
**Tech Stack:** React (Vite), Google ADK (Python), Cloud Run, Firestore

## 1. Product Vision

Stop overthinking and start doing. Decidely.ai uses a swarm of AI agents to chat with you, research options via MCP, and weigh the pros and cons. It’s like having a personal board of advisors to help you make the right call, stress-free.

## 2. User Persona

**Alex (The Overthinker):** Alex is trying to buy a new piece of tech but is overwhelmed by YouTube reviews and spec sheets. Alex needs someone to ask the right questions, filter the noise, and present a logical conclusion with a bit of moral support.

## 3. User Journey & Workflow

- **Initiation:** User opens the web app and types a dilemma: "I need to pick a destination for my anniversary trip, budget is $2000."
- **Clarification (Interviewer):** The Primary Agent routes this to the Interviewer agent, which replies: "Congratulations! Do you prefer the beach or the mountains? Any specific dates?"
- **Research (Researcher):** Once criteria are gathered, the system silently triggers the Researcher agent to use Google Search Grounding to find flights and hotels fitting the $2000 budget.
- **Analysis (Evaluator):** The Evaluator agent uses the SQLite MCP tool to create a database table, insert the destinations, and score them based on the user's preferences (e.g., Beach = +10 points).
- **Recommendation & Support:** The Primary Agent reads the SQLite output, formulates the final recommendation, and the Supporter agent adds an empathetic closing remark. The result is displayed to the user along with the "Decision Matrix" table.
- **Data Storage:** The entire session and structured matrix are saved to Google Cloud Firestore.

## 4. Functional Requirements

### 4.1. Frontend (React/Vite)
- **Chat Interface:** Standard message bubbles distinguishing between User and "Board of Directors" (System).
- **Agent Status Indicator:** A visual indicator showing which agent is currently "thinking" (e.g., "Researcher is gathering data...", "Evaluator is crunching numbers...").
- **Matrix Viewer:** A dedicated UI component to render the structured decision matrix (Pros/Cons/Scores) returned by the Evaluator.

### 4.2. Backend API (Python/FastAPI)
- **RESTful Endpoints:**
  - `POST /api/chat`: Accepts user messages and session ID.
  - `GET /api/history/{session_id}`: Retrieves past decisions from Firestore.
- **Agent Orchestration (Google ADK):** Implement a Supervisor pattern. The Primary Agent must have the authority to decide the next step (Reply to user OR route to sub-agent).

### 4.3. Agent Capabilities & Tools

| Agent Name | Role | Associated Tools / MCPs |
| :--- | :--- | :--- |
| **Primary** | Orchestrator & final communicator | Multi-agent routing logic |
| **Interviewer** | Extracts constraints and preferences | None (LLM reasoning only) |
| **Researcher** | Gathers external facts and options | Google Search Grounding |
| **Evaluator** | Scores options against user criteria | SQLite MCP (for structuring matrices) |
| **Supporter** | Analyzes sentiment, provides encouragement | None (LLM reasoning only) |
| **Primary (Stretch Goal)** | Export final report | Google Drive MCP |

### 4.4. Database (Firestore)
- **Collections:**
  - `sessions`: Stores the chat transcript.
  - `decisions`: Stores the structured JSON outcome of the decision matrix for future user reference.

## 5. Non-Functional Requirements

- **Performance:** The backend must return a stream or a status update within 3 seconds so the frontend doesn't appear frozen while agents talk to each other.
- **Deployment:** Backend must be containerized (Dockerfile) and deployed to Google Cloud Run. Frontend must be deployed to Vercel or Firebase Hosting.
- **Cost Efficiency:** Must rely strictly on scale-to-zero serverless architecture to remain under the $5/month budget constraint.

## 6. Out of Scope (Hackathon MVP)

- User account creation (OAuth).
- Voice input/output.
- Real-time multi-user collaboration on a single decision.

## 7. Stretch Goals

- **Knowledge Export:** Integrate the Google Drive MCP so the Primary Agent can save the final "Decision Matrix" and summary report directly to the user's Google Drive as a document.
