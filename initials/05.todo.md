# Hackathon Implementation To-Do List: Decidely.ai

This step-by-step guide is designed to take you from an empty folder to a fully deployed MVP, keeping the $5 budget and hackathon constraints in mind.

## Phase 1: Environment & Infrastructure Setup

- [ ] Initialize Monorepo: Create the base folder structure (as outlined in folder_structure.md).
- [ ] GCP Project Setup:
  - [ ] Create a new Google Cloud Project.
  - [ ] Enable billing (required for APIs, but you'll stay in the free tier).
  - [ ] Enable the Vertex AI API (for Gemini / Google ADK).
  - [ ] Enable Cloud Run API.
  - [ ] Create a Firestore Database in Native Mode.
- [ ] Backend Setup (Python):
  - [ ] Set up a virtual environment (uv venv).
  - [ ] Install dependencies: uv pip install fastapi uvicorn google-adk google-cloud-firestore pydantic mcp.
  - [ ] Create .env file for GCP credentials and API keys.
- [ ] Frontend Setup (React):
  - [ ] Initialize Vite project: bun create vite@latest frontend --template react.
  - [ ] Install Tailwind CSS and configure tailwind.config.js.
  - [ ] Install API client: bun add axios.

## Phase 2: Core Backend & Database (API Foundation)

- [ ] Firestore Integration:
  - [ ] Create backend/app/core/firestore.py.
  - [ ] Write functions to save_session(session_id, messages) and save_decision_matrix(session_id, matrix_data).
- [ ] FastAPI Setup:
  - [ ] Create backend/app/main.py.
  - [ ] Define the Pydantic schemas in models/schemas.py for request/response bodies.
  - [ ] Create the base POST /api/chat and GET /api/history/{session_id} routes (return mock data for now to test connection).

## Phase 3: Multi-Agent System (The Core Engine)

- [ ] Setup Base Agents (Pure LLM):
  - [ ] Create Interviewer agent with prompts focused on asking clarifying questions.
  - [ ] Create Supporter agent with prompts focused on empathy and sentiment analysis.
- [ ] Setup Tool-Equipped Agents:
  - [ ] Create Researcher agent. Attach Google Search Grounding tool.
  - [ ] Implement the SQLite MCP Client.
  - [ ] Create Evaluator agent. Give it access to the SQLite MCP to create tables, insert researched options, and run scoring queries.
- [ ] Build the Orchestrator:
  - [ ] Create the Primary agent.
  - [ ] Define the routing logic (Supervisor pattern) using google-adk. Give the Primary agent instructions on when to call the Interviewer vs. Researcher vs. Evaluator.

## Phase 4: Frontend UI Development

- [ ] Chat Interface:
  - [ ] Build the main ChatInterface.jsx component.
  - [ ] Implement message bubbles distinguishing "User" from "Board of Directors".
- [ ] Agent Status Indicator:
  - [ ] Build a dynamic loader component (e.g., "Researcher is gathering data...") that updates based on backend API streams or status flags.
- [ ] Matrix Viewer:
  - [ ] Build DecisionMatrix.jsx to parse and render the JSON output from the SQLite MCP into a clean HTML table (Pros, Cons, Total Score).
- [ ] API Integration:
  - [ ] Connect the frontend to the FastAPI POST /api/chat endpoint.

## Phase 5: End-to-End Integration & Testing

- [ ] Workflow Testing:
  - [ ] Run a test conversation: Input a prompt -> Check if Primary routes to Interviewer -> Answer criteria -> Check if Researcher searches -> Check if Evaluator scores -> Check final output.
- [ ] Database Verification:
  - [ ] Verify that chat transcripts and the final decision matrices are successfully saving to Firestore.
- [ ] Refine Prompts:
  - [ ] Tweak system prompts to prevent agents from getting stuck in loops. Ensure the Primary agent knows how to synthesize the final message.

## Phase 6: Deployment

- [ ] Backend (Cloud Run):
  - [ ] Write the Dockerfile for the FastAPI backend.
  - [ ] Build and push the container to Google Artifact Registry.
  - [ ] Deploy to Cloud Run (ensure unauthenticated invocations are allowed for the API, set minimum instances to 0 to save money).
- [ ] Frontend (Vercel / Firebase Hosting):
  - [ ] Update frontend API calls to point to the live Cloud Run URL instead of localhost.
  - [ ] Run bun run build.
  - [ ] Deploy the dist folder to Vercel or Firebase Hosting.
  - [ ] Final QA: Run a full test on the live URLs.

## Phase 7: Stretch Goals (Optional)

- [ ] Google Drive MCP Integration:
  - [ ] Set up Google Drive API credentials.
  - [ ] Implement `backend/app/mcp/drive_client.py`.
  - [ ] Give the Primary Agent the ability to call the Drive MCP to export the final decision report.
