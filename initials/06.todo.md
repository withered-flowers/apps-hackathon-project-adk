# Hackathon Implementation To-Do List: Decidely.ai

This step-by-step guide is designed to take you from an empty folder to a fully deployed MVP, keeping the $5 budget and hackathon constraints in mind.

## Phase 1: Environment & Infrastructure Setup

- [x] Initialize Monorepo: Create the base folder structure (as outlined in folder_structure.md).
- [x] GCP Project Setup:
  - [ ] Create a new Google Cloud Project.
  - [ ] Enable billing (required for APIs, but you'll stay in the free tier).
  - [ ] Enable the Vertex AI API (for Gemini / Google ADK).
  - [ ] Enable Cloud Run API.
  - [ ] Create a Firestore Database in Native Mode.
- [x] Backend Setup (Python):
  - [x] Set up a virtual environment (`uv venv` and activate it with `source .venv/bin/activate.fish`).
  - [x] Install dependencies: uv pip install fastapi uvicorn google-adk google-cloud-firestore pydantic mcp.
  - [x] Create .env file for GCP credentials and API keys.
- [x] Frontend Setup (React):
  - [x] Initialize Vite project: bun create vite@latest frontend --template react.
  - [x] Install Tailwind CSS and configure tailwind.config.js.
  - [x] Install API client: bun add axios.

## Phase 2: Core Backend & Database (API Foundation)

- [x] Firestore Integration:
  - [x] Create backend/app/core/firestore.py.
  - [x] Write functions to save_session(session_id, messages) and save_decision_matrix(session_id, matrix_data).
- [x] FastAPI Setup:
  - [x] Create backend/app/main.py.
  - [x] Define the Pydantic schemas in models/schemas.py for request/response bodies.
  - [x] Create the base POST /api/chat and GET /api/history/{session_id} routes (return mock data for now to test connection).

## Phase 3: Multi-Agent System (The Core Engine)

- [x] Setup Base Agents (Pure LLM):
  - [x] Create Interviewer agent with prompts focused on asking clarifying questions.
  - [x] Create Supporter agent with prompts focused on empathy and sentiment analysis.
- [x] Setup Tool-Equipped Agents:
  - [x] Create Researcher agent. Attach Google Search Grounding tool.
  - [x] Implement the SQLite MCP Client.
  - [x] Create Evaluator agent. Give it access to the SQLite MCP to create tables, insert researched options, and run scoring queries.
- [x] Build the Orchestrator:
  - [x] Create the Primary agent.
  - [x] Define the routing logic (Supervisor pattern) using google-adk. Give the Primary agent instructions on when to call the Interviewer vs. Researcher vs. Evaluator.

## Phase 4: Frontend UI Development

- [x] Chat Interface:
  - [ ] Build the main ChatInterface.jsx component.
  - [x] Implement message bubbles distinguishing "User" from "Board of Directors".
- [x] Agent Status Indicator:
  - [x] Build a dynamic loader component (e.g., "Researcher is gathering data...") that updates based on backend API streams or status flags.
- [x] Matrix Viewer:
  - [ ] Build DecisionMatrix.jsx to parse and render the JSON output from the SQLite MCP into a clean HTML table (Pros, Cons, Total Score).
- [x] API Integration:
  - [ ] Connect the frontend to the FastAPI POST /api/chat endpoint.
